# Position Assistant - 持仓助手

基于价值投资理念的港股筛选工具，结合 PB 估值和九神指数（ahr999）进行仓位管理。

## 功能特性

- 📊 **港股筛选**：基于 PE、PB、股息率、市值等指标筛选港股
- 🏭 **智能行业分类**：按 13 个大类归并（地产/建筑/银行/保险/运输/电力/电信/医药/消费/科技/工业/资源/新能源），支持手动映射表修正
- 📈 **双维度评分**：
  - 前景评分（100分）：ROE、毛利率、营收增长、现金流、负债率
  - 分红评分（100分）：连续分红年数、派息率、增长趋势
- 💰 **仓位建议**：
  - 股票仓位：基于平均 PB 的 7 档建议
  - 数字货币仓位：基于九神指数（ahr999）的 7 档建议
- 📄 **HTML 报告**：生成美观的筛选报告

## 快速开始

### 1. 安装依赖

```bash
pip install pandas requests
```

### 2. 获取 API Token

访问 [理杏仁开放平台](https://www.lixinger.com/open/api) 注册并获取 API Token。

### 3. 配置

**方式一：环境变量（推荐）**

```bash
export LIXINGZHE_TOKEN="your_api_token_here"
export OUTPUT_DIR="./reports"
export HTTP_PROXY=""  # 可选
```

**方式二：配置文件**

```bash
cp scripts/config.example.json scripts/config.json
# 编辑 config.json 填入你的配置
```

### 4. 运行

```bash
python3 scripts/position_assistant.py
```

## 筛选流程

```
┌─────────────────────────────────────────────────────────────┐
│                    港股筛选流程 (14步)                       │
└─────────────────────────────────────────────────────────────┘

Step 1: 获取港股列表
    └── 调用理杏仁 API 获取全部港股公司数据

Step 2: 获取基本面数据
    ├── 按财报类型分类（非金融/银行/保险/证券/REIT）
    └── 批量获取 PE、PB、股息率、市值等指标

Step 3: 合并数据
    └── 将公司信息与基本面数据合并

Step 4: 初筛（估值过滤器）
    ├── 市值 ≥ 50 亿港元
    ├── 0.01 ≤ PE ≤ 15
    ├── 0.01 ≤ PB ≤ 1.5
    └── 排除指定股票代码

Step 5: 行业分类
    ├── 优先级1: 手动映射表（最可靠）
    ├── 优先级2: API 返回的行业名称
    ├── 优先级3: 关键词匹配（公司名+主营业务）
    ├── 优先级4: 财报类型区分金融子类
    └── 兜底: 归类为"工业"

Step 6: 排除本地银行
    └── 过滤掉规模较小的城商行/农商行

Step 7: 基础打分排序
    ├── PE 排名（升序）
    ├── PB 排名（升序）
    ├── 股息率排名（降序）
    └── 综合得分 = 三项排名之和

Step 8: 选出 TOP 150
    └── 按综合得分取前150只作为候选池

Step 9: 详细打分（双维度）
    ├── 前景评分（满分100分）
    │   ├── ROE水平（20分）
    │   │   └── ≥20%得20分，≥15%得16分，≥10%得12分...
    │   ├── ROE稳定性（10分）
    │   │   └── 5年标准差<3%得10分，<6%得7分...
    │   ├── ROE趋势（8分）
    │   │   └── 连续3年增长且为正得8分
    │   ├── 净利率趋势（8分）
    │   ├── 净利润CAGR（10分）
    │   │   └── 4年复合增长≥15%得10分
    │   ├── 营收CAGR（10分）
    │   │   └── 4年复合增长≥15%得10分
    │   ├── 现金流覆盖（10分）
    │   │   └── OCF/净利润≥1.2倍得7分
    │   ├── 毛利率水平+趋势（14分）
    │   └── 负债率（10分）
    │       └── ≤30%得10分，≤45%得8分...
    │
    └── 分红评分（满分100分）
        ├── 连续分红年数（30分）
        │   └── ≥10年得30分，≥7年得25分...
        ├── 派息率合理性（30分）
        │   └── 20%-70%区间得30分（最健康）
        ├── 分红增长趋势（25分）
        │   └── 连续3年增长得25分
        └── 可持续性（15分）
            └── 派息率在健康区间得15分

Step 10: 综合排序
    ├── 综合得分 = 前景分 + 分红分
    └── 按综合得分降序排列

Step 11: 评级划分
    ├── 按前景分与分红分形成综合排序
    ├── 前景标签与分红标签分开展示
    └── 最终结果支持按类别观察

Step 12: 分类筛选 TOP N
    ├── 按综合得分遍历
    ├── 使用 13 个用户定义大类
    ├── 每类最多选 3 家
    └── 目标总数 = 13 × 3 = 39（不足则按实际符合条件数量输出）

Step 13: 仓位计算
    ├── 获取九神指数（ahr999）
    │   └── 从非小号 API 获取实时数据
    ├── 计算平均 PB
    │   └── 最终入选股票池的平均市净率
    ├── 股票仓位建议（基于PB）
    │   ├── PB < 0.60 → 股票80% : 现金20%
    │   ├── 0.60-0.85 → 股票70% : 现金30%
    │   ├── 0.85-1.15 → 股票60% : 现金40%
    │   ├── 1.15-1.45 → 股票50% : 现金50%
    │   ├── 1.45-1.75 → 股票40% : 现金60%
    │   ├── 1.75-2.00 → 股票30% : 现金70%
    │   └── PB ≥ 2.00 → 股票20% : 现金80%
    └── BTC仓位建议（基于ahr999）
        ├── < 0.21 → BTC80% : 现金20%
        ├── 0.21-0.315 → BTC75% : 现金25%
        ├── 0.315-0.49 → BTC70% : 现金30%
        ├── 0.49-0.84 → BTC60% : 现金40%
        ├── 0.84-1.26 → BTC45% : 现金55%
        ├── 1.26-1.75 → BTC30% : 现金70%
        └── > 1.75 → BTC20% : 现金80%

Step 14: 生成报告
    ├── 输出筛选结果摘要
    ├── 生成 HTML 报告
    └── 保存到指定目录
```

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| ROE水平 | 20分 | ≥20%(20) ≥15%(16) ≥10%(12) ≥5%(6) ≥0%(2) |
| ROE稳定性 | 10分 | 标准差<3%(10) <6%(7) <10%(4) <15%(1) |

| 净利率趋势 | 8分 | 连续3年增长(8) |
| 净利润CAGR | 10分 | ≥15%(10) ≥8%(7) |

## OpenClaw 集成

可以作为 OpenClaw 的 Skill 使用，支持定时任务自动运行。

### Cron Job 配置示例

```json
{
  "name": "position-assistant",
  "schedule": {"kind": "cron", "expr": "0 3 * * *", "tz": "Asia/Shanghai"},
  "sessionTarget": "isolated",
  "payload": {
    "kind": "agentTurn",
    "message": "Run position assistant: python3 ~/.openclaw/skills/position-assistant/scripts/position_assistant.py",
    "timeoutSeconds": 1200
  }
}
```
